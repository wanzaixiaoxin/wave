# 造物运输大亨 — v1.1 体验优化方案

> **来源**：体验评审报告 + 产品侧补充分析  
> **目标**：在不改变核心架构的前提下，把"玩起来的感觉"提升一个档次  
> **原则**：改动最小化，效果最大化

## 详细设计

### O-01：首屏引导 + 任务式新手流程

#### 当前问题
玩家打开游戏看到空页面 + 一堆数字，不知道第一步做什么。

#### 设计方案

```
状态 A：首次进入（无存档）
─────────────────────────────
• 页面加载后，车库区域显示一个带箭头的引导浮层：
  "👉 点击这里造你的第一辆车"
  箭头指向 [🔧 造车] 按钮，呼吸动画
  
• 造出第一辆车后：
  弹出命名框（提供 3 个随机名字备选）
  命名完成后提示："💡 等几秒订单刷新后，点击「派车」"

• 成功派单后提示：
  "💡 车跑订单时可以研究科技或管理工厂"

• 以上提示只在首次出现，不重复

状态 B：再次进入（有存档）
─────────────────────────────
• 正常显示，无引导
• 如果有离线收益，弹窗展示
```

#### 实现方式
- 用 `localStorage` 标记 `tutorial_completed`
- 引导浮层用纯 CSS 动画（无需额外库）
- 箭头指向用 `position: relative` + 偏移计算

### O-02：金币飘字动画 + 顶栏数值跳动

#### 当前问题
订单完成后金币数字直接跳变，玩家感受不到"赚钱了"的快感。

#### 设计方案

```
① 订单完成飘字
   触发：每完成一个订单
   动画：从当月卡片位置（或屏幕中央区域）向上飘起
         "+200🪙" 字号 24px 金色粗体
         向上移动 60px + 淡出，持续 1.2 秒
         CSS animation + @keyframes

② 顶栏金币跳动
   触发：金币数值变化
   动画：数字短暂放大 1.3 倍 → 恢复
         持续 0.3 秒
         CSS transition transform

③ 暴击特效（概率触发）
   触发：暴击时（收入 ×2 或 ×3）
   动画：金色闪光 + 大字 "💥 暴击！×2"
         屏幕中央放大 → 缩小 → 消失，持续 1.5 秒
```

#### 实现方式
- 在 `renderOrders` 中监听到 `ORDER_COMPLETED` 事件时
- 动态创建飘字 DOM 元素，插入到 `#app`，自动消失
- 顶栏金币元素的 class 切换 `gold-bounce`

### O-03：品质视觉差异化

#### 当前问题
白/蓝/金品质只靠一个 2px 的边框颜色区分，视觉上缺乏"稀有感"。

#### 设计方案

```
白板 ⚪：灰色边框，普通卡片
精良 🔵：蓝色边框 + 左上角蓝色小标签 "精良"
传说 🟡：金色边框 + 金色微光动画（box-shadow pulse）
        左上角金色小标签 "传说"
        卡片背景轻微金色渐变（rgba 叠加）

标签实现：
  .quality-badge {
    position: absolute; top: 4px; right: 4px;
    padding: 1px 6px; border-radius: 4px;
    font-size: 10px; font-weight: bold;
  }
```

### O-04：车辆卡片信息分层

#### 当前问题
每张卡片塞了 10 种信息，全部 11px 灰色小字。

#### 设计方案

```
第一层（始终显示）：
  ┌──────────────┐
  │    🛴        │  ← 大 emoji
  │ 小轮子       │  ← 名字，14px bold
  │ ✅ 空闲      │  ← 状态标签，彩色背景
  │ [品质标签]   │  ← 小角标
  └──────────────┘

第二层（鼠标悬浮/点击后看到）：
  弹窗/详情模式显示完整信息：
  等级、品质、特质、亲密度、4 维属性、战绩
```

### O-05：订单可派车辆直接显示

#### 当前问题
订单上只显示"有车可用 / 无空闲车辆"，玩家不知道具体哪辆车可以派。

#### 设计方案

```
📮 普通配送  +200🪙
经验 10
🚗 可派: 小轮子 ✅  |  [派车]

如果有多辆可用车：
🚗 可派: 小轮子 +2  |  [派车]
（表示有 3 辆车可用，系统自动选最优的）

如果没有可用车：
🔴 暂无空闲车辆  |  [派车]（灰色不可点，点击提示原因）
```

### O-06：订单完成通知

#### 当前问题
订单完成只有日志里一行字，玩家可能看不到。

#### 设计方案

```
在屏幕右上角弹出一个 Toast 通知：
┌──────────────────┐
│ ✅ 小轮子 完成订单 │    ← 3 秒后自动消失
│    +200🪙 +10经验 │
└──────────────────┘

实现：
  • 固定定位 top: 20px right: 20px
  • CSS animation slide-in-right → fade-out
  • 同时最多显示 3 条，超出覆盖
```

### O-07：科技树可点击提示增强

#### 当前问题
可研究的科技节点不够明显，玩家可能不知道可以点。

#### 设计方案

```
可研究的节点：
  • 按钮上显示 "🔬 研究" 文字按钮（不只是改光标）
  • 脉冲边框动画（box-shadow pulse）
  • 鼠标悬停时显示研究条件

不可研究的节点：
  • 清晰显示缺少什么
    缺金币："❌ 需要 800🪙"
    缺条件："🔒 需要先产 5 辆马车"
    已完成："✅ 已完成"
```

### O-08：顶栏增加车辆状态图标

#### 当前问题
玩家切换到工厂/科技 Tab 后，看不到车库车辆的运行状态。

#### 设计方案

```
顶栏右侧增加两个小图标：
  🟢 3 — 空闲车辆数（绿色）
  🚚 2 — 派单中车辆数（橙色）

在任何 Tab 下都可以看到有多少车在赚钱、多少车闲着。
如果需要操作车辆，点击图标快速切回车库 Tab。
```
