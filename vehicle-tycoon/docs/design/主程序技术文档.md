### 1.3 存档存储

```typescript
const SAVE_KEY = 'tycoon_save_v1';
const SAVE_INTERVAL = 30; // 自动保存间隔（秒）

// 存档大小预估：约 15-30 KB（含 JSON 序列化）
// 序列化 + 压缩后约 8-15 KB

function saveGame(data: SaveData): void {
  const json = JSON.stringify(data);
  // 可选：LZString 压缩
  // const compressed = LZString.compress(json);
  localStorage.setItem(SAVE_KEY, json);
}

function loadGame(): SaveData | null {
  const raw = localStorage.getItem(SAVE_KEY);
  if (!raw) return null;
  // 可选：LZString 解压缩
  // const decompressed = LZString.decompress(raw);
  return JSON.parse(raw);
}
```

## 第二章：游戏循环架构

### 2.1 主循环

```
游戏主循环使用 requestAnimationFrame，每秒固定 tick 60 次。
经济系统使用 setInterval，每秒 tick 1 次。

requestAnimationFrame（60fps）—— UI 渲染、动画、输入响应
       │
       └→ updateUI()
       └→ handleInput()
       └→ renderAnimations()
       
setInterval（1 次/秒）—— 经济计算、生产进度、订单倒计时
       │
       └→ tickEconomy()
            ├→ FactorySystem.tick(1)     // 产线生产
            ├→ VehicleSystem.tick(1)     // 车辆状态更新
            ├→ OrderSystem.tick(1)       // 订单倒计时
            ├→ EventSystem.tick(1)       // 随机事件判定
            └→ AchievementSystem.check() // 成就判定
```

### 2.2 经济系统 tick（每秒执行）

```typescript
function tickEconomy(deltaSeconds: number): void {
  // 1. 产线生产
  for (const line of factory.productionLines) {
    if (line.currentOrder) {
      line.currentOrder.remainingTime -= deltaSeconds;
      if (line.currentOrder.remainingTime <= 0) {
        onProductionComplete(line.index);
      }
    }
  }

  // 2. 车辆订单进度
  for (const vehicle of garage.vehicles) {
    if (vehicle.status === 'on_order') {
      // 订单进度由 OrderSystem 管理
    }
  }

  // 3. 自动保存
  autoSaveCounter += deltaSeconds;
  if (autoSaveCounter >= SAVE_INTERVAL) {
    autoSaveCounter = 0;
    saveGame(getCurrentSaveData());
  }
}
```

### 2.3 离线收益计算

```typescript
function calculateOfflineEarnings(offlineSeconds: number): OfflineResult {
  const MAX_OFFLINE = 2 * 3600; // 最多 2 小时
  const OFFLINE_EFFICIENCY = 0.4; // 40% 效率
  
  const effectiveSeconds = Math.min(offlineSeconds, MAX_OFFLINE);
  const effectiveTicks = effectiveSeconds * OFFLINE_EFFICIENCY;
  
  // 假设所有产线持续生产 T1（离线收益保守估算）
  let totalGold = 0;
  let totalParts = 0;
  let totalCars = 0;
  
  for (const line of factory.productionLines) {
    if (line.currentOrder) {
      const tier = line.currentOrder.tier;
      const config = getVehicleConfig(tier);
      const carsProduced = Math.floor(effectiveTicks / config.buildTime);
      
      totalCars += carsProduced;
      totalGold += carsProduced * config.basePrice;
      totalParts += carsProduced * config.partsCost * 0.1; // 零件产出
    }
  }
  
  return {
    offlineSeconds: effectiveSeconds,
    carsProduced: totalCars,
    goldEarned: Math.floor(totalGold),
    partsEarned: Math.floor(totalParts),
  };
}

// 玩家上线时调用
function onPlayerLogin(): void {
  const now = Date.now();
  const saveData = loadGame();
  const offlineSeconds = (now - saveData.timestamp) / 1000;
  
  if (offlineSeconds > 10) { // 超过 10 秒视为离线
    const result = calculateOfflineEarnings(offlineSeconds);
    applyOfflineEarnings(result);
    showOfflineDialog(result);
  }
}
```

## 第四章：性能目标

| 指标 | 目标 | 说明 |
|------|------|------|
| 初始加载时间 | < 3 秒 | 首屏可交互 |
| 存档大小 | < 50 KB | localStorage 配额通常 5MB |
| 同时渲染车辆 | 20+ 辆 | @60fps |
| 动画同时播放 | 5+ 个 | @30fps |
| 离线计算 | < 100ms | 最多离线 2 小时的计算 |
| 内存占用 | < 100MB | 含所有纹理和音频 |
| 电池消耗 | 后台 tick 不开启 | 只在激活窗口时执行 tick |

## 第六章：AI 开发遵循规范

### 6.1 代码生成要求

```
1. 所有 TypeScript 代码使用 strict 模式
2. 所有函数必须有完整的 JSDoc 注释（参数/返回值/异常）
3. 数值配置必须从 VehicleConfig 表读取，不允许硬编码
4. 存档读写必须通过 SaveManager 单例
5. 事件监听必须在组件销毁时取消注册
6. 所有 UI 文案必须从 i18n 配置读取（中英文双语言）
```

### 6.2 测试要求

```
覆盖以下测试场景：
1. 核心循环测试：模拟 1000 次 tick，验证经济不崩
2. 离线测试：离线 2 小时，验证收益计算正确
3. 边界测试：车库满时造新车、金币为负时操作、同时 10 辆车派单
4. 存档测试：保存 → 修改存档 → 读取存档 → 验证数据完整性
5. 跨版本测试：v1.0 存档被 v1.1 读取时的兼容性
```

### 6.3 分支规范

```
main          → 稳定版本，只合入经过测试的代码
develop       → 日常开发分支
feature/*     → 功能分支（feature/challenge-mode）
bugfix/*      → 修复分支（bugfix/save-corruption）
release/*     → 发布准备分支（release/v1.0）
```
