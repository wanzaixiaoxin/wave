## 二、界面问题与改进建议

### 问题 1：首屏信息过载，玩家不知道第一步该做什么

**现状**：打开游戏看到的是——顶栏 5 个数字、一个下拉框 + 2 个按钮、车库区域（空的）、订单区域（空的）、底栏 4 个 Tab。玩家的视线不知道该落在哪里。

```
┌─ 顶栏: 🪙 ⚙️ 💖 0/s T1 ─────────────────┐
├─ [▼选车型] [🔧造车] [🏠扩建]              │
├─ 🏠 车库 (0/6)                            │
│     "车库是空的，点击造车"                  │
├─ 📋 订单                                  │
│     "先造一辆车，订单会自动出现"             │
├─ [🏠车库] [🏭工厂] [🔬科技] [🏆成就]       │
└────────────────────────────────────────────┘
→ 玩家的视线在 7 个区域之间跳跃，不知道该先看哪里
```

**建议**：

```
① 视觉引导：第一个车辆卡片添加呼吸光效或箭头指引
   "👉 点击这里造你的第一辆车"

② 简化首屏：首次进入时，非核心区域（工厂/科技/成就）折叠隐藏，
   只显示：顶栏 + 造车按钮 + 车库

③ 任务式引导：第一步不是自由探索，而是"造一辆独轮车→给它起名→派它接单"
   完成一步才解锁下一步的提示
```

### 问题 2：车辆卡片信息太密，没有视觉层次

**现状**：每张卡片塞了 emoji + 名字 + 等级 + 品质 + 特质 + 进化状态 + 亲密度 + 订单数 + 收入 + 状态标签。10 种信息挤在 140px 宽的卡片里，全部是 11px 灰色小字。

```
┌──────────────┐
│    🛴        │
│  小轮子       │
│ Lv.5/10 | 🟡  │  ← 3 种信息挤在一行
│ 🌟已进化 💖45 │  ← 又是 2 种
│ 📦12单|🪙3.4k│  ← 又是 2 种
│   ✅ 空闲     │
└──────────────┘
```

**建议**：

```
① 分级展示：
   第一眼（最重要的）：emoji + 名字 + 状态（空闲/派单中）
   点击后看到的：等级、品质、特质、亲密度、战绩

② 品质视觉差异化：
   金品质：卡片背景微光 + 金色名字
   蓝品质：蓝色边框
   白品质：灰色（默认）

③ 状态标签突出：
   "空闲"用绿色大标签 ✅
   "派单中"用车在路上的小动画 🚚……
```

### 问题 3：缺少"钱在增加"的视觉快感

**现状**：订单完成后，金币数字跳一下。没有任何视觉反馈告诉玩家"你赚钱了"。

**建议**：

```
① 金币飘字动画：订单完成时，从车辆位置飘出 "+200🪙" 飞向顶栏
   （即使没有动画引擎，也可以用 CSS 动画模拟：从下往上 + 淡出）

② 顶栏数字跳跃动画：金币变化时数字短暂放大闪烁

③ 订单完成提示：弹一个小通知"XX 完成订单 +200🪙"（3 秒自动消失）
   而不是只写在日志里
```

### 问题 4：订单区域与车库的视觉关系不清晰

**现状**：订单在车库下方，但看起来像两个独立的模块。玩家不容易理解"派车 = 从车库选一辆车 → 派去执行订单"这个流程。

**建议**：

```
① 视觉连接：点击订单的"派车"按钮时，高亮显示当前可用的车辆
   （可用车辆边框闪烁）

② 简化操作链：在订单卡片上直接显示可派车辆的名字
   "📮 普通配送 +200🪙
    🚗 可用: 小轮子 ✅"
   而不是只显示"有车可用"

③ 订单完成后，车辆卡片上出现一个小动画（车回来的效果）
```

### 问题 5：标签页切换后丢失上下文

**现状**：切换到工厂/科技/成就 Tab 后，玩家看不到车库和订单了。如果想看车跑完订单没有，必须切回车库。

**建议**：

```
① 顶部悬浮栏：在任何 Tab 下都显示：
   金币变化 + 订单完成数 + 一个"车辆状态"小图标（绿=空闲/黄=派单中）

② 快捷通知：当车辆完成订单回到车库时，
   即使你在工厂 Tab，也弹一个小通知

③ 或者：把订单区域固定在底部，不随 Tab 切换
```

### 问题 6：科技树的"可点击"不够明显

**现状**：可研究的科技节点用绿色左边框和普通 cursor 标识，但玩家可能不知道可以点击。

**建议**：

```
① 可点击的节点添加脉冲动画或"研究"按钮
② 不可点击的节点清晰显示缺什么（缺金币/缺零件/条件未满足）
③ 当前科技名称旁边显示一个小进度条
```

## 三、玩法问题与改进建议

### 问题 7：前 30 秒没有"游戏感"

**现状**：玩家打开游戏，看到一片数字。没有目标、没有紧迫感、没有"想玩下去"的冲动。

**建议**：

```
① 开场一个小目标："造出你的第一辆车！"（3 秒自动完成）
   让玩家第一秒就有事做，而不是看空页面

② 造车动画：点击造车后，看到一个简单的"零件组装→闪光→完成"动画
   （即使是 CSS 动画：旋转 fade in）

③ 给第一辆车命名时，提供几个随机有趣名字作为备选：
   "小旋风 / 闪电号 / 风之子 / 铁蛋"
   比让玩家从零想一个名字更友好
```

### 问题 8：核心循环缺少"涨落感"

**现状**：游戏循环是平的——一直造车、一直派单、一直在涨。没有低谷和高潮的区分。

**建议**：

```
① 加入"等待→收获"的节奏：
   派单后车辆出去跑 → 玩家看着它倒计时 → 回来时一波收入
   现在的订单完成太快（30 秒），几乎没有"期待感"
   建议：前期订单 30 秒 OK，但高级订单延长到 60-90 秒

② 加入小惊喜：
   订单完成时概率触发"暴击！收入 ×2"的大字特效
   特质"幸运"触发时：金色闪光 + 特殊音效提示

③ 品质提升/升级时：
   不要只是数字变一下
   显示 "⭐ 品质提升！白板→精良" + 车辆边框变色动画
```

### 问题 9：车辆之间缺少差异化感受

**现状**：独轮车和自行车的玩法完全一样——都是造出来、派单、升级。换车型只是数值更高，没有"手感不同"。

**建议**：

```
① 差异化的订单体验：
   独轮车只能接近距离单（收入低但快）
   卡车可以接重货单（收入高但慢）
   火车需要等凑够一批货才出发（批量收入）
   → 这样选车型不只看数值，还看"运营风格"

② 车型特殊交互：
   点击自行车：铃铛声 🔔
   点击汽车：引擎轰鸣
   点击轮船：汽笛声
   （现阶段用 emoji 文字代替声音）

③ 车型养成可视化：
   进化后的车在卡片上多一个"进化标记"（🌟或✨）
   不同品质的车拥有不同的卡片背景色
```

### 问题 10：缺少"成长感"的量化反馈

**现状**：玩家知道金币在涨，但不知道自己"变强了多少"。

**建议**：

```
① 添加"里程碑时刻"：
   总收入达到 1,000 时：🎉"你已经赚到第一桶金了！"
   造出第 10 辆车时：🎉"你的车队初具规模！"
   解锁 T4（小汽车）时：🎉"进入内燃机时代！"
   这些在成就系统里已有，但触发时要有全屏小动画

② 时间线/成长图：
   一个简单的"你的帝国"总览页：
   累计收入、总车辆数、最高车型、游戏时长

③ 车库"荣誉墙"：
   退役的车辆在纪念册中显示
   不只是数据，而是"XX 号车辆，服役期间完成了 XX 单，
   总收入 XX，于 XX 日退役"
   让每辆退役的车都有"墓志铭"
```

## 四、优先级排序

```
🔴 紧急（不修玩家会流失）
────────────────────────
□ 首屏引导缺失 —— 玩家不知道第一步做什么
  → 最简单的修法：加一个"👉 点击造车"的指向性提示
□ 金币获取无视觉反馈 —— 玩家感受不到"赚钱了"
  → 最简单的修法：CSS 飘字动画 "+200🪙"

🟡 重要（影响中期体验）
────────────────────────
□ 核心循环缺少涨落感 —— 节奏太平
  → 延长高级订单时间 + 暴击特效
□ 车辆间无差异化 —— 换车型=换皮
  → 车型特有道具类型 + 专属交互反馈
□ 标签页切换丢失上下文 —— 看不到车辆状态
  → 顶栏增加车辆状态小图标

🟢 建议（体验优化）
────────────────────────
□ 科技树可点击提示不足
□ 品质升级视觉反馈不足
□ 成长感量化反馈（里程碑动画）
□ 退役车辆"墓志铭"
```

---

## 五、一句话总结

> **核心循环（造车→派单→赚钱→升级）是对的，但玩家感受不到这个循环的"快感"。每笔钱到账没有反馈、每次升级没有仪式感、每辆车没有个性。游戏逻辑上跑通了，体验上还差一口气。**

如果只改一件事：**给金币收入加上飘字动画**。这是投入产出比最高的改动——让玩家看到钱在涨，是放置游戏的原始快感。
