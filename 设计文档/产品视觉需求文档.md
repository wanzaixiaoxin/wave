# 造物运输大亨 — 产品视觉需求文档 v1.0

> **本文档定位**：主策划输出的产品级视觉要求，定义游戏"看起来是什么样、玩起来是什么感觉"。  
> **下游文档**：主美根据本文档的风格方向输出 `主美美术文档.md` 的具体资源规范。  
> **核心问题**：这个游戏长什么样？玩家第一眼看到什么？玩的时候视觉重心在哪？

---

## 一、核心视觉定位

### 1.1 一句话定义

> **"一个温暖的、可以捧在手心里的车库世界。"**

### 1.2 三个视觉关键词

```
🏠 归属感          🚗 生命力          ☕ 陪伴感
（像个家）        （车是活的）        （像喝杯茶的轻松）
```

### 1.3 风格决策：2D 还是 3D？

```
结论：纯 2D 渲染，不引入 3D。

理由：
1️⃣ 情感传递效率 —— 2D 扁平插画的表情/颜色传递情感比 3D 模型更直接
2️⃣ 开发成本 —— 没有 3D 建模/绑定/灯光/材质管线，团队可以更小
3️⃣ 迭代速度 —— 2D 资源产出快，可以快速试错替换
4️⃣ 风格差异化 —— 市面上的模拟经营很多走 3D 低多边，2D 扁平反而有辨识度
5️⃣ 性能 —— 2D 比 3D 少 5-10 倍的性能开销，覆盖更多低端设备

例外情况：车辆详情页的"车辆展示"可以做一个伪 3D 效果
  —— 即 2D 多帧动画模拟旋转视角（8 帧不同角度），不是真 3D
```

## 二、场景设计

### 2.1 场景清单

整个游戏只有 3 个场景，没有场景加载（都在一个界面内切换）：

| 场景 | 名称 | 作用 | 视觉重心 |
|------|------|------|----------|
| 🏠 | **车库**（主场景） | 车辆停放、养成操作、情感连接 | 车辆卡片墙 |
| 🏭 | **工厂**（副场景） | 生产管理、科技树、扩建 | 产线流水线 |
| 🌌 | **荣誉殿堂**（收藏场景） | 退役车辆展示、成就、纪念 | 灯光聚焦的老车 |

**不需要**：世界地图、户外场景、战斗场景、剧情过场。

### 2.2 车库场景（主场景，玩家 70% 时间在这里）

```
视觉描述：

背景是一面暖灰色的车库墙壁，有轻微的混凝土纹理（用 2D 噪点模拟）。
墙上有几盏暖黄色的壁灯，投下柔和的扇形光线。
地面是浅灰色的环氧地坪，有微弱的反光。

车辆不是停在地上，而是停在"展示台"上——每个展示台是一个
浅木色的方形底座，底部有柔和的阴影，让车看起来像是
"被精心陈列的收藏品"。

壁灯的光线照射在展示台上，形成暖黄色的光晕。
车库深处（屏幕两侧）渐暗，营造"还有很多车停在那里"的想象空间。

右上角有一扇小窗户，可以看到外面是傍晚的天空（随时间变化）。
```

**氛围参考**：
```
颜色参考：#2C3E50（墙壁）+ #ECF0F1（地面）+ #F39C12（灯光）
质感参考：木质展示台、混凝土墙面、金属壁灯
情绪：安静、温暖、像是自己家的车库
```

### 2.3 工厂场景（玩家 20% 时间在这里）

```
视觉描述：

与车库相同的建筑风格，但空间更开阔。
左侧是产线区域——传送带上正在组装车辆（用简化的 2D 动画表示）。
右侧是控制面板——显示产线状态和队列。
背景比车库更亮，体现"工作区域"的氛围。

传送带上的车辆零件用几何色块表示（轮子=圆、车身=矩形），
装配时逐步拼接完成，最后"叮"一声整车下线。

工厂升级后，视觉变化：
  Lv.1：手工作坊——木质工具台、简单吊灯
  Lv.3：机械工厂——传送带、机械臂（2D 动画）
  Lv.5：自动化工厂——全流水线、灯光更亮、效率感
```

### 2.4 荣誉殿堂场景（玩家 10% 时间在这里，情感场景）

```
视觉描述：

深色背景（深蓝灰 #1A252F），聚光灯从上方打下。
每辆退役车辆悬浮在独立的圆形光柱中。
光柱颜色随车辆品质变化：
  白：冷白光  蓝：暖蓝光  金：暖金光

车辆缓慢自转（2D 多帧动画模拟）。
下方显示车辆名字、等级、一段自动生成的回忆文字：
  "这辆独轮车陪我完成了最初的 47 单订单。"
  "它是我的第一辆传说品质车。"

背景有非常微弱的星光粒子缓慢飘落。
```

## 三、玩法形态与界面布局

### 3.1 屏幕布局（固定，不随页面切换改变）

```
┌──── 状态栏（安全区域）────────────────────┐
│  🪙 123,456    ⚙️ 234    💖 45           │ ← 始终显示
├────────────────────────────────────────────┤
│                                            │
│          主内容区域（可变）                   │
│    ┌──────────────────────────────┐       │
│    │                              │       │
│    │   车库 / 工厂 / 殿堂 切换     │       │
│    │                              │       │
│    └──────────────────────────────┘       │
│                                            │
├────────────────────────────────────────────┤
│  🏠车库  🔬科技  🏭工厂  🏆成就  📖纪念册 │ ← 底栏 Tab
└────────────────────────────────────────────┘
```

### 3.2 信息层级（优先级从高到低）

```
第一眼看到（0.5 秒内）：
  1. 金币数量（顶栏左侧）—— 最核心资源
  2. 车辆卡片（主区域）—— 核心内容

第二眼看到（1-2 秒内）：
  3. 可操作的按钮（派单/升级/造车）—— 下一步行动
  4. 订单区域（底部）—— 当前任务
  5. 零件数量（顶栏中间）—— 次要资源

需要时看到（用户主动查找）：
  6. 亲密度数值
  7. 成就进度
  8. 科技树状态
  9. 工厂产线详情
  10. 纪念册/退役记录
```

### 3.3 触控操作规范

```
由于是手机优先，所有交互必须适配手指触控：

点击区域最小 44×44pt（Apple HIG 标准）
滑动操作：水平滑动切换车辆卡片
长按：显示车辆快捷信息预览（不进入详情页）
下拉：刷新订单列表

不用：
  ❌ 悬停（Hover）—— 手机上不存在
  ❌ 双指手势 —— 不符合休闲游戏用户习惯
  ❌ 3D Touch —— 仅限部分设备

支持：单手操作，拇指可达区域（屏幕下半部分）放置所有操作按钮
```

### 3.4 布局安全区

```
以 iPhone 12（390×844pt）为设计基准：

顶部安全区：60pt（刘海/状态栏）
底部安全区：40pt（Home Indicator）
底栏高度：60pt（Tab 按钮）
顶栏高度：48pt（资源显示）

主内容区域：844 - 60(安全) - 48(顶栏) - 60(底栏) - 40(安全) = 636pt
```

## 四、玩法形态定义

### 4.1 游戏的"玩法镜头"

```
场景：固定俯视角度，不旋转，不缩放。
视角高度：略高于平视（15度俯角），可以看到车辆正面和展示台。

不提供：
  ❌ 自由视角旋转
  ❌ 拉近拉远缩放
  ❌ AR 模式
  ❌ VR 模式

理由：这是休闲挂机游戏，不是 3A 大作。固定视角降低认知负担，
玩家不需要管理"镜头在哪里"，专注于做决策。
```

### 4.2 核心玩法对应视觉

| 玩法操作 | 视觉表现 |
|----------|----------|
| 造新车 | 产线上零件拼装→闪光→新车出现在车库 |
| 派单 | 从车库滑出一辆车→驶向屏幕外→车轮滚动动画 |
| 收单 | 车从屏幕外驶回→金币飘向顶栏 |
| 升级属性 | 车辆弹跳+数值放大闪烁 |
| 品质提升 | 边框从白→蓝→金色渐变过渡 |
| 进化 | 全屏粒子聚集→重组为新形态 |
| 退役告别 | 车辆缓缓上升→光柱特效→进入殿堂 |
| 离线归来 | 屏幕淡入→数字滚动显示离线收益 |

### 4.3 界面中的"情感时刻"

特意设计的视觉情感触点：

```
情感时刻 1：给新车起名
  触发：造完第一辆车（且仅限第一辆，后续自动取名）
  视觉：车辆特写放大，背景变暗，输入框柔和居中
  目的：让玩家认真对待"这是属于我的车"

情感时刻 2：首次告别
  触发：第一次送走车辆
  视觉：车库灯光变暖，车辆缓缓驶向屏幕上方，
        屏幕上浮现一行字："谢谢你陪我走过的路。"
  目的：让告别有仪式感，不是冷冰冰的删除

情感时刻 3：进化
  触发：第一次进化
  视觉：全屏白光闪烁→车辆新形态展示→名字焕新
        全服公告滚动（如果联网）
  目的：里程碑高潮，玩家最强烈的成就感来源

情感时刻 4：离线回归
  触发：每次上线（离线超过 30 分钟）
  视觉：车库灯光渐亮，车辆从暗到明，
        弹出离线报告："你的车们想你了❤️"
  目的：让离线回归也温暖，而不是冷冰冰的数据
```

## 五、产品级别技术约束

### 5.1 设备覆盖

```
目标设备：
  iOS 13+     ：iPhone 8 及以上
  Android 8+  ：主流中低端机型
  屏幕比例    ：16:9 到 19.5:9 自适应
  最低内存    ：2GB RAM
  最低分辨率  ：750×1334（iPhone 6 基准）
```

### 5.2 性能基准（产品要求）

```
帧率：目标 60fps，最低 30fps
安装包大小：目标 < 50MB（含所有资源）
最大纹理尺寸：2048×2048
同时精灵数：< 200（一个场景内）
Draw Call：< 100
```

### 5.3 无障碍基础要求

```
• 所有按钮有文字标签（不仅仅是图标）
• 颜色不作为唯一信息传递方式（色盲友好）
  （例如：品质除了颜色外，也显示"白/蓝/金"文字）
• 字号最小 12pt
• 对比度 ≥ 3:1（文字与背景）
```

## 七、附录：竞品视觉分析简表

| 竞品 | 风格 | 可借鉴 | 不要学 |
|------|------|--------|--------|
| 旅行青蛙 | 温暖手绘风、低饱和度 | 色彩体系、情感化设计 | 信息密度太低 |
| 开罗游戏 | 像素风、高信息密度 | 卡片布局、UI 效率 | 像素风不符合本作调性 |
| 企鹅岛 | Q 版 3D、圆润 | 角色表情、收集展示 | 3D 技术路线 |
| 月兔冒险 | 2D 扁平、暖色调 | 场景氛围 | 玩法太被动 |
